# Security Policy

## Reporting a vulnerability

Please do **not** open public issues for suspected vulnerabilities.

Instead, report privately to project maintainers through the repository's private security contact channel. Include:

- description of the issue
- steps to reproduce
- affected version/commit
- potential impact
- suggested mitigation (if known)

## Response process

Maintainers will:

1. acknowledge receipt
2. investigate and assess severity
3. prepare and validate a fix
4. coordinate disclosure timing

## Disclosure

We prefer coordinated disclosure after a fix is available.

## Security best practices for contributors

- avoid committing secrets/credentials
- validate and sanitize external inputs
- add tests for security-sensitive changes
- keep dependencies up to date
